Numa AI safety

Numa is guide-first. Numa can help users correct errors, spot missing context, and choose the next action. Numa cannot change code, rewrite names, mutate database records, or reveal private money, deal, salary, compensation, budget, margin, or contract details unless the user is Administrator-unlocked or has the Super user role.

Default users Guide-only. Sensitive financial and CRM deal details are redacted. Numa recommends app actions instead of changing records.
Super users Change-enabled. Assign the Super user role below to allow Numa to discuss sensitive operational details and propose controlled updates.
CRM focus On CRM, Numa prioritizes which relationships or deals need attention by urgency, risk, stalled follow-up, next meeting, and relationship health.
To activate full Numa privileges for a person, edit that user and set Role to Super user. Administrator access also unlocks full Numa privileges after the Admin password check.
Numa change mode is off.
